# TODO LIST

本阶段目标：实现多服务账号首页、Emby / Jellyfin 服务首页、媒体详情页；下一步重点推进现代化 UI 细节、i18n、明暗主题、服务卡片拖拽排序和 libmpv 播放器接入。

当前约束：

- 当前已有 Emby / Jellyfin 登录、媒体库与库内列表的基础实现，后续新增代码仍需保持最小修改和分层边界。
- 后续编码继续按 TODO 拆分任务、阅读现有实现并查阅对应官方文档。
- 后续实现前必须查阅 Emby / Jellyfin 官方 API 文档，禁止凭经验猜接口。
- QML 只负责页面展示、交互和导航；登录、网络请求、媒体库解析等逻辑放在 C++ 层。
- 网络请求统一使用 `QNetworkAccessManager`。
- Token、Cookie、密码等敏感信息禁止写入日志。
- 测试服务器凭据只允许作为本地临时验证信息使用，禁止写入仓库、文档、日志或截图。
- 服务卡片、详情页、播放页等 UI 不得直接展示密码、Token、Cookie 等敏感信息。
- UI 后续需要建立统一视觉规范，所有按钮、输入框、弹窗、卡片、列表、菜单等控件都要现代化，整体质感参考现代浏览器中的组件库体验，例如 Ant Design。
