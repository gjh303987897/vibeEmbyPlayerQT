# WebDAV 模块说明

## 模块边界

WebDAV 下载相关功能拆成五层：

- `WebDavClient`：标准 WebDAV 请求与 XML 解析。
- `WebDavItemListModel`：目录条目模型与显示模式过滤。
- `CredentialStore`：WebDAV 密码的系统凭据存储。
- `WebDavDownloadPlanner`：单次遍历远程目录并生成本地下载计划。
- `TransferManager`：运行期后台上传/下载队列与进度统计。
- `AppViewModel`：服务卡片、目录导航、上传下载入口、播放入口的编排。

QML 只负责展示和交互，不直接发网络请求。

WebDAV 页面使用独立的中文文案表覆盖上传、下载、下载任务、目录加载和存储空间提示，未命中的键再回退到全局翻译表。

## 目录显示模式

WebDAV 文件页提供三种显示模式：

- 默认模式：保留原有列表布局，展示当前目录返回的全部子文件夹和文件。
- 视频模式：只展示当前目录中的子文件夹，以及由 C++ 层识别为可播放视频的文件；其他普通文件不会进入 QML 模型。
- 音频模式：只保留当前目录中的音频文件，并立即按目录列表中的既有顺序开始播放。音频队列不递归进入子文件夹。

视频模式使用缩略图卡片网格：

- 子文件夹显示文件夹缩略图和文件夹名称。
- 视频显示视频缩略图、扩展名、文件名和文件大小。
- 点击子文件夹继续沿用现有 WebDAV 目录导航；点击视频继续沿用现有播放代理；卡片下载按钮继续使用现有下载规划和传输队列。
- 切换模式只重建本地可见条目，不重新发送 `PROPFIND`，进入新的子文件夹后会继续应用当前模式。
- 默认列表和视频网格都在固定内容区域内滚动并使用 Qt Quick 视图虚拟化，目录条目较多时不会一次性创建全部卡片。

音频模式的业务状态由 `AppViewModel` 管理：

- `WebDavClient` 根据标准 `getcontenttype` 的 `audio/*` MIME 类型与常见音频扩展名标记音频文件；`WebDavItemListModel` 只负责本地模式过滤。
- 音频队列复制当前目录中标记为音频的条目，保留 `WebDavClient` 已按名称排好的顺序。切换模式不重新发送 `PROPFIND`。
- 播放每一首时继续使用 `WebDavPlaybackProxy`，WebDAV 凭据不会暴露给 QML 或写入播放 URL 日志。
- libmpv 正常播放到文件末尾后自动播放下一首；单个文件加载失败时记录错误并继续下一首。最后一首结束后停止队列并返回 WebDAV 文件页。
- QML 音乐播放器只展示当前曲目、进度、暂停、上一首、下一首、音量和播放队列，并把用户操作转发给 C++ ViewModel，不在 QML 中计算或维护队列。

标准 WebDAV `PROPFIND` 不定义视频画面预览地址。当前缩略图由 QML 本地绘制为文件夹/视频视觉卡片，不额外读取远程视频数据，避免目录浏览触发大量媒体流量或阻塞 UI。未来如增加真实画面缩略图，应单独实现跨平台异步缩略图缓存服务，不得在 QML 中下载或解析视频。

## 协议实现

当前使用标准 WebDAV 方法：

- `PROPFIND` + `Depth: 1`：列出当前目录。
- `PROPFIND` + `Depth: 0`：获取单个资源信息。
- `MKCOL`：创建远程目录。
- `GET`：下载文件。
- `PUT`：上传文件。

认证使用 Qt 的 `QNetworkAccessManager::authenticationRequired`，由 `QAuthenticator` 处理 Basic/Digest。

## 凭据策略

SQLite 不保存 WebDAV 密码。

当前策略：

- Windows：使用 Credential Manager。
- 其他平台：返回不可用，自动降级为每次输入密码。

后续如果增加 macOS/Linux 支持，应继续放在 `CredentialStore` 内，不要把平台代码泄漏到 ViewModel 或 QML。

## 传输任务

`TransferManager` 是运行期队列：

- 任务类型：上传、下载、远程建目录。
- 支持进度、状态、单文件/总任务暂停与恢复、失败或取消重试、取消。
- 每次用户发起的下载都会生成一个顶层总任务；单文件下载同样按一个总任务展示。
- 文件夹中的实际文件作为总任务的子任务保存，下载页默认只展示总任务，进入总任务后才展示逐文件进度。
- 文件明细页提供全部、未完成、已完成、失败、已取消五种等宽筛选；未完成包含等待中、传输中和已暂停状态。
- 总任务暂停会暂停其全部排队和运行子任务；单文件暂停只影响对应子任务，其他文件继续按并发队列执行。
- Qt `QNetworkReply` 没有可保留当前 GET 连接的暂停接口。运行中下载暂停时调用 `abort()`、关闭并删除不完整文件；恢复后从该文件开头重新下载。排队中的文件暂停不会发起网络请求。
- 重试只重新排队状态为失败或已取消的文件，已经完成的文件不会重复下载；总任务的重试按钮会重试该任务中的全部失败或已取消文件，文件明细的重试按钮只重试对应文件。
- 单文件取消会关闭活动写入并删除对应本地文件；总任务取消会移除排队任务、中止运行任务，并删除该任务已产生的全部本地文件，文件夹下载会清理本次规划得到的独立目标目录。
- 取消后的下载定义会保留到用户清除已结束任务，因此完成本地清理后仍可重做；重试时会重新创建已被总任务取消删除的目标目录。
- 第一版不支持 HTTP Range 断点续传，不跨应用重启恢复暂停任务。
- 文件夹上传会把远程目录创建和文件上传按队列执行。
- 上传文件任务支持暂停、继续、取消和重新上传。运行中暂停会中止当前 `PUT` 并释放本地只读文件，继续时从文件开头重新上传；排队中的上传可直接暂停而不发起请求。
- 取消或上传失败后会保留本地源文件与任务定义，用户可在清除已结束任务前重新上传。远程建目录任务不提供暂停，取消上传也不会删除本地源文件。
- 第一版不支持上传断点续传或分块续传协议，暂停后的上传会完整重传当前文件。
- 下载任务最多同时执行 3 个；上传和远程建目录保持独占串行，避免改变文件夹上传顺序。
- 多文件下载使用一次批量模型插入，进度更新只刷新对应行，并限制为约每 120 毫秒发布一次。
- 传输连续 60 秒没有数据或进度时会超时；失败或取消的下载会删除不完整文件，上传任务始终保留本地源文件。
- 下载页使用虚拟化 `ListView`，不会因任务数增加而一次性创建全部 QML 行。

下载统计口径：

- 当前速度：按下载与上传两个方向分别展示；各方向为正在运行任务的瞬时采样速度之和，采样周期约 500 毫秒。
- 平均速度：按下载与上传两个方向分别展示；单个总任务从首个文件开始，以累计字节数除以实际经过时间。
- 剩余下载量：所有子任务可确认总大小时，以总字节数减去已完成字节数；任一文件大小未知或累加溢出时显示“计算中”。
- 顶层待完成、已完成、失败/取消计数按总任务统计，不会把一个文件夹中的每个文件重复计数。
- 下载统计行按固定权重分配列宽，列宽只随页面宽度变化，不受计数、速度或剩余大小文本变化影响。

`TransferTaskListModel` 分别承载顶层任务和当前选中总任务的文件明细。QML 通过 ViewModel 暴露的选择接口切换模型，不持有或计算下载业务状态。

统计实现遵循 Qt 的 `QNetworkReply::downloadProgress` / `uploadProgress` 约定：无法预知的 `bytesTotal` 为 `-1`，收到可确认总量后再更新剩余下载量。平均速度使用 `QElapsedTimer` 的单调计时结果计算，暂停期间不计入总任务平均速度时间。上传通过 `QNetworkAccessManager::put(request, QIODevice*)` 读取本地文件；Qt 官方接口确认 `abort()` 会中止操作并仍触发 `finished()`：<https://doc.qt.io/qt-6/qnetworkreply.html#downloadProgress>、<https://doc.qt.io/qt-6/qnetworkreply.html#uploadProgress>、<https://doc.qt.io/qt-6/qnetworkreply.html#abort>、<https://doc.qt.io/qt-6/qnetworkaccessmanager.html#put>、<https://doc.qt.io/qt-6/qelapsedtimer.html>。

## 文件夹下载规划

`WebDavDownloadPlanner` 使用标准 `PROPFIND` + `Depth: 1` 逐层遍历文件夹：

- 最多并发读取 4 个远程目录。
- 在同一次遍历中同时收集文件、空目录和总大小，不再先估算大小再重复扫描下载内容。
- 对远程名称进行本地文件名清理，并为重名文件或目录预留唯一目标路径。
- 规划完成后先创建本地目录，再把全部文件一次性加入下载队列。

RFC 4918 允许客户端使用一系列 `Depth: 1` 请求遍历集合，以兼容拒绝 `Depth: infinity` 的服务器：<https://www.rfc-editor.org/rfc/rfc4918>。

## 播放

WebDAV 视频和音频播放都通过本地 `WebDavPlaybackProxy` 把远程内容交给 libmpv。视频使用窗口嵌入；音频使用不创建视频窗口、但保留音频输出的 headless libmpv 实例。

需要认证时，播放代理通过 `QNetworkAccessManager` 和 `QAuthenticator` 访问远端，libmpv 只读取本机回环地址。QML 和 libmpv 都不会接触 WebDAV 密码。

注意：

- 不要记录密码到日志。
- 离开播放时清空临时播放凭据。

## 下载安全检查

下载前会检查目标磁盘空间：

- 单文件使用 `getcontentlength`。
- 文件夹在下载规划的同一次递归 `PROPFIND` 中累计总大小。
- 若空间不足，弹警告但允许继续。
- 若大小无法确认，弹警告但允许继续。

本地重名文件使用自动重命名，规则为：

- `file.ext`
- `file (1).ext`
- `file (2).ext`

文件夹同理。
