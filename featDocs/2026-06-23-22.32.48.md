# 播放器内 IPTV 频道列表

## 背景

用户在观看 IPTV 时，需要能够直接在播放器中打开频道列表并切换频道，避免退出播放器回到 IPTV 页面重新选择。

本次需求同时要求弹出界面与播放器已有控制层保持一致，并避免重新引入之前的视频播放器遮挡、黑屏、窗口层级和 mpv 加载问题。

## 实现内容

### 1. 播放器内新增 IPTV 频道入口

在播放器底部控制栏中新增 `频道 / Channels` 按钮。

按钮仅在当前播放内容是 IPTV 频道时显示：

- IPTV 播放时显示。
- Emby / Jellyfin / WebDAV / 本地验证播放时不显示。

判断依据由 C++ 层提供：

- `AppViewModel::iptvPlaybackActive()`
- `AppViewModel::currentIptvChannelId()`

### 2. 新增右侧 IPTV 频道浮层

播放器页面新增 `playerIptvChannelWindow`。

该浮层延续播放器已有设计：

- 使用独立 `Window`，不放回 `MpvVideoItem` 内部。
- 深色半透明背景。
- 10px 圆角。
- 与字幕、音轨、倍速菜单一致的边框、hover、按下、选中状态。
- 右侧面板布局，不遮挡底部进度条和核心播放按钮。
- 打开浮层后停止自动隐藏控制栏。
- 支持 `Esc` 关闭。
- 点击视频区域会关闭浮层并显示控制栏。

浮层内包含：

- 标题与当前过滤后的频道数量。
- IPTV 搜索框，复用 `appViewModel.iptvSearchText`。
- 分组横向列表，复用 `appViewModel.iptvGroups` 与 `selectIptvGroup(...)`。
- 频道列表，复用 `appViewModel.iptvChannels`。
- 当前频道高亮，并显示 `播放中 / On air`。
- 频道 logo；没有 logo 时显示频道首字母占位。

### 3. 切台复用现有播放入口

点击频道时继续调用已有 C++ 入口：

```cpp
AppViewModel::playIptvChannel(int row)
```

没有新增第二套播放逻辑。

该入口仍负责：

- 校验频道 URL。
- 更新 `currentPlaybackUrl`。
- 清理非 IPTV 播放会话字段。
- 更新当前选中媒体信息。
- 发出 `playbackChanged()`。
- 切换到 `player` 视图。

`MpvVideoItem::setSource(...)` 已确认在 `currentPlaybackUrl` 改变时会触发 `play()`，因此播放器内切台会重新加载新频道。

### 4. 当前频道定位

`IptvChannelListModel` 新增：

```cpp
Q_INVOKABLE int indexOfChannelId(const QString& channelId) const;
```

打开频道浮层时，QML 使用该接口定位当前频道，并将列表滚动到当前项附近。
