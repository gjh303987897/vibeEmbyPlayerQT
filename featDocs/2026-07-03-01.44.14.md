# 播放器直连视频流量补偿统计

## 背景

历史统计页面已经可以统计 Qt 网络层可见的 API 请求、WebDAV 目录请求、WebDAV 上传下载和 WebDAV 播放代理流量。

用户实测发现，播放在线视频并拖动进度条后，系统任务管理器显示实际网络流量达到数百 MB，但历史统计页面只记录到约 244.2 KB。

复查后确认原因是：Emby、Jellyfin 和 IPTV 的正片播放 URL 会直接交给 libmpv，真实视频下载、seek 后的 Range 请求和 HLS 分片请求都由 libmpv 发起，不经过 `QNetworkAccessManager`，因此旧统计只能看到少量 API 流量。

本次新增“播放器直连视频流量补偿统计”，用于把 libmpv 直连视频输入流量纳入历史统计。

- `cache-speed` 表示 cache 与底层输入层之间的当前 I/O 读取速度，单位为 bytes/s，client API 类型是 `MPV_FORMAT_INT64`。

- `raw-input-rate` 是网络层或其他字节输入层的估算输入速率，可能不精确或缺失。

参考：

## 实现内容

### 1. PlayerController 采样 libmpv 输入速度

在 `PlayerController` 中新增：

- `QTimer m_networkStatsTimer`
- `QElapsedTimer m_networkSampleElapsed`
- `bool m_networkStatsActive`
- `playbackNetworkBytes(qint64 bytesReceived)` 信号
- `sampleNetworkStats()` 私有方法

播放请求成功后启用采样：

```cpp
m_networkStatsActive = true;
m_networkSampleElapsed.restart();
```

停止播放、播放结束和关闭播放器时关闭采样：

```cpp
m_networkStatsActive = false;
```

每秒读取一次 mpv 属性：

```cpp
mpv_get_property(m_mpv, "cache-speed", MPV_FORMAT_INT64, &bytesPerSecond);
```

再根据真实计时间隔换算本次采样字节数并发出信号。

### 2. MpvVideoItem 转发播放器流量信号

`MpvVideoItem` 新增：

```cpp
void playbackNetworkBytes(qint64 bytesReceived);
```

并转发 `PlayerController::playbackNetworkBytes`。

该改动只增加信号转发，不修改 native window 创建、同步、显示、隐藏、DPR 换算或 Z 序逻辑。

### 3. QML 播放器实例上报采样值

在 `qml/Main.qml` 的播放器实例中新增：

```qml
onPlaybackNetworkBytes: function(bytesReceived) {
    appViewModel.recordPlaybackNetworkBytes(bytesReceived)
}
```

QML 只负责把播放器信号转给 ViewModel，不在 QML 中实现统计业务逻辑。

### 4. AppViewModel 接入现有历史统计聚合

`AppViewModel` 新增：

```cpp
Q_INVOKABLE void recordPlaybackNetworkBytes(qint64 bytesReceived);
```

实现中会：

- 忽略非正数字节数。
- 根据当前播放上下文找到当前服务。
- WebDAV 服务跳过播放器层采样。
- 复用已有 `recordNetworkUsage()` 写入历史统计 pending buffer。

因此以下能力自动保持一致：

- 按天、按服务聚合。
- 30 天保留。
- 历史页打开时刷新 pending 数据。
- 隐私模式下的记录按隐私统计写入，非隐私模式历史页不显示隐私统计。

## WebDAV 去重策略

WebDAV 播放本来就通过 `WebDavPlaybackProxy` 统计远端响应数据块。

libmpv 播放 WebDAV 时读取的是本地代理 URL：

```text
http://127.0.0.1:<port>/<streamId>/stream
```

如果同时统计 mpv 的 `cache-speed`，会把本机回环读取也算进去，导致重复统计。

因此 `recordPlaybackNetworkBytes()` 对 `ServiceType::WebDAV` 直接返回，WebDAV 仍由播放代理统计真实远端流量。

## 注意

mpv 官方文档说明 `cache-speed` / `raw-input-rate` 是估算值，可能不精确或缺失。

因此历史统计中的播放器直连流量用于补齐视频本体拉流量级，目标是避免 KB 级严重偏低；它不等同于操作系统任务管理器的网卡统计值。

任务管理器可能还包含：

- 协议开销。
- 系统或其他进程流量。
- DNS、TLS、TCP 等额外字节。
- 本机回环或驱动层差异。
