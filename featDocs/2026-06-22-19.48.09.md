# 播放器缓存统计

## 需求

在播放器已有播放/暂停、进度条、加载提示和视频信息检测之外，增加美观的缓存统计，显示当前视频大约已经缓存了多少秒。

## 修改前复查

实现前复查了播放器历史问题：

- `bugFix/2026-06-21-15.11.08.md`
  - 避免改动 `loadfile` 参数顺序和播放开始上报时机。
- `bugFix/2026-06-21-15.48.45.md`
  - 继续使用独立工具窗口和 `raiseChromeWindows()` 管理播放器浮层，避免被 Windows native mpv popup HWND 遮挡。
- `bugFix/2026-06-22-14.35.14.md`
  - 不改动 WebDAV 本地播放代理、认证转发和 Range 请求处理。
- `featDocs/2026-06-22-18.31.26.md`
  - 复用视频信息面板的展示结构和 QML 属性转发方式。

本次参考 mpv 官方手册的 `demuxer-cache-duration` 属性。该属性表示 demuxer 中大约缓存的视频秒数，部分媒体或播放状态下可能不可用。

## C++ 实现

### `PlayerController`

新增只读属性：

- `cacheDurationSeconds`

新增信号：

- `cacheStatsChanged()`

实现方式：

- 在 `observeProperties()` 中观察 mpv 属性：
  - `demuxer-cache-duration`
- 通过 `handlePropertyChange()` 接收缓存秒数变化。
- 使用 `-1.0` 表示未知或不可用状态。
- 当 mpv 返回 `MPV_FORMAT_NONE` 时清空为未知，避免显示旧缓存数值。
- 播放新 URL、播放结束、进入 idle、停止播放时同步清空缓存秒数。
- 变化小于 `0.1` 秒时不重复发信号，降低 UI 抖动和刷新频率。

### `MpvVideoItem`

新增 QML 可绑定属性：

- `cacheDurationSeconds`

并转发 `PlayerController::cacheStatsChanged()` 给 QML。

## QML 实现

### 底部控制条

在播放器底部控制条中新增缓存状态胶囊：

- 显示 `缓存 12s` / `Cache 12s`。
- 超过 60 秒时显示为 `1m 05s` 这类短格式。
- 如果 mpv 暂时没有可用缓存秒数，显示 `--`。
- 胶囊使用暗色背景和浅色文字，尺寸受限，避免挤压音量控件。

### 视频信息面板

复用上一轮新增的 `playerInfoWindow`：

- 增加 `已缓存 / Cached` 一行。
- 面板高度从 `236` 调整为 `264`，容纳第五项信息。
- 继续使用独立工具窗口，不新增新的浮层窗口，减少焦点和层级复杂度。

## 文案

新增英文文案：

- `player.cacheDuration`
- `player.cacheShort`

新增中文文案：

- 已缓存
- 缓存

并将 `player.infoHint` 调整为说明信息来自 mpv 播放与缓存状态。

## 避坑说明

本次没有修改：

- `PlayerController::playUrl()` 的 `loadfile` 参数结构。
- Emby / Jellyfin 播放 URL 生成。
- WebDAV 本地播放代理。
- WebDAV Basic / Digest 认证转发。
- WebDAV 自签名证书处理。
- Windows native mpv popup HWND 创建、DPR 换算和 Z 序逻辑。
- 现有播放/暂停、进度条、字幕、音轨、倍速、音量和加载提示逻辑。

缓存统计只是读取 mpv 的只读属性并展示，不参与播放控制链路。

## 验证重点

- 本地视频播放仍能正常输出画面。
- 控制条可以显示缓存秒数或 `--`。
- 视频信息面板可以显示缓存秒数或 `--`。
- 缓存统计不可用时不影响播放。
- 播放切换、停止或退出后不会残留上一条视频的缓存秒数。
