# 剧集/季支持实现记录

## 需求

在视频简介页面增加电视剧、纪录片等多季多集媒体的浏览能力，并同时支持 Emby 与 Jellyfin。

## 官方 API 依据

- Jellyfin 官方 OpenAPI `https://api.jellyfin.org/openapi/jellyfin-openapi-stable.json`
  - `GET /Shows/{seriesId}/Seasons`，operationId 为 `GetSeasons`
  - `GET /Shows/{seriesId}/Episodes`，operationId 为 `GetEpisodes`
  - Episodes 支持 `seasonId`、`userId`、`fields`、`enableImages`、`enableUserData`、`sortBy`
- Emby 官方 REST API `https://dev.emby.media/reference/RestAPI/TvShowsService.html`
  - `GET /Shows/{Id}/Seasons`，说明为获取 tv series 的 seasons
  - `GET /Shows/{Id}/Episodes`，说明为获取 tv season 的 episodes
  - Episodes 页面确认支持 `SeasonId`、`UserId`、`Fields`、`EnableImages`、`EnableUserData`、`StartIndex`、`Limit`、`SortBy`、`SortOrder`

## 代码改动

### 服务层接口

文件：

- `src/services/media/MediaServiceClient.h`

新增接口：

- `fetchSeriesSeasons(const UserSession&, const QString& seriesId, ...)`
- `fetchSeasonEpisodes(const UserSession&, const QString& seriesId, const QString& seasonId, ...)`

这保持了 QML -> ViewModel -> Service -> Network 的分层，QML 不直接访问网络。

### Emby 支持

文件：

- `src/services/emby/EmbyClient.h`
- `src/services/emby/EmbyClient.cpp`

实现：

- Seasons: `GET /Shows/{Id}/Seasons`
- Episodes: `GET /Shows/{Id}/Episodes`
- 查询参数使用 Emby PascalCase：`UserId`、`SeasonId`、`Fields`、`EnableImages`、`EnableUserData`、`SortBy`、`SortOrder`
- 请求字段包含封面、简介、评分、运行时长、用户播放进度等详情页需要的数据。

### Jellyfin 支持

文件：

- `src/services/jellyfin/JellyfinClient.h`
- `src/services/jellyfin/JellyfinClient.cpp`

实现：

- Seasons: `GET /Shows/{seriesId}/Seasons`
- Episodes: `GET /Shows/{seriesId}/Episodes`
- 查询参数使用 Jellyfin camelCase：`userId`、`seasonId`、`fields`、`enableImages`、`enableUserData`、`sortBy`
- 复用通用 JSON 解析，不在 Jellyfin 客户端中写重复解析逻辑。

### 通用模型与解析

文件：

- `src/models/MediaItem.h`
- `src/services/media/MediaServerClientBase.cpp`
- `src/viewmodels/MediaItemListModel.h`
- `src/viewmodels/MediaItemListModel.cpp`

改动：

- `MediaItem` 新增 `childCount`，用于季卡片显示集数。
- Series 类型条目如果服务端没有返回 `SeriesId`，会回退使用自身 `Id`，便于后续加载 seasons。
- `MediaItemListModel` 新增 `childCount` role。
- seasons 和 episodes 仍复用 `MediaItem`，因为 Emby/Jellyfin 返回的季和集都是 BaseItemDto 形态。

### ViewModel 状态流

文件：

- `src/viewmodels/AppViewModel.h`
- `src/viewmodels/AppViewModel.cpp`

新增 QML 属性：

- `selectedItemIsSeries`
- `selectedSeasonId`
- `selectedSeasonName`
- `seriesSeasons`
- `seriesEpisodes`

新增方法：

- `selectSeason(int row)`
- `openEpisode(int row)`
- `loadSeriesSeasons()`
- `loadSeasonEpisodes(const MediaItem& season)`
- `clearSeriesDetails()`

流程：

1. 打开媒体详情后，ViewModel 先加载基础详情。
2. 如果详情条目类型是 `Series`，自动请求 seasons。
3. seasons 返回后默认选择第一个 season。
4. 选择 season 后请求该 season 下的 episodes。
5. 点击 episode 后进入该集详情页，继续复用原有播放 URL、断点续播和播放上报流程。

为了避免用户快速切换页面时旧请求覆盖新页面，ViewModel 使用 `m_seriesRequestGeneration` 对 season/episode 请求做代际校验。

### QML 展示

文件：

- `qml/Main.qml`

改动：

- Series 详情页隐藏原来的直接播放按钮，避免对 Series 调用播放 URL。
- 增加 `Seasons & Episodes` 区域。
- 增加横向 season pill 列表。
- 增加 episode grid，展示封面、集名、季集号、时长、简介和观看进度。
- 点击 episode 调用 `appViewModel.openEpisode(index)`，进入该集详情页。

新增 QML 组件：

- `SeasonPill`
- `EpisodeCard`
