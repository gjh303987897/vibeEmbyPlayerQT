# 历史统计页面

## 背景

用户需要在服务页顶部的 `添加`、`编辑`、`设置` 前增加 `历史统计` 入口，用于查看过去一个月内每天的播放记录、每个服务的观看时长以及网络流量使用情况。

本次实现目标：

- 在首页顶部新增 `历史统计` 按钮。
- 新增独立历史统计页面，视觉上与当前深色媒体中心界面一致。
- 统计数据按天、按服务聚合。
- 数据保留 30 天，过期记录自动删除。
- 避免触碰播放器窗口嵌入、mpv `loadfile` 参数、WebDAV 播放代理认证与 Range 转发等历史高风险路径。

## 实现内容

### 1. 首页顶部新增入口

在服务页顶部工具栏新增 `历史统计` 按钮，位置位于：

`历史统计` → `添加` → `编辑` → `设置`

点击后调用：

```cpp
AppViewModel::openHistoryStats()
```

该入口只在服务页显示，返回按钮会从历史统计页回到服务页。

### 2. 新增历史统计页面

新增 `HistoryPage`，包含：

- 30 天保留说明。
- 总观看时长卡片。
- 总网络流量卡片。
- 按日期分组的每日服务记录。
- 空状态卡片。
- 手动刷新按钮。

页面复用现有主题色：

- `theme.elevated`
- `theme.elevatedHover`
- `theme.input`
- `theme.border`
- `theme.primary`
- `theme.success`
- `theme.text`
- `theme.muted`

没有引入新的视觉体系。

### 3. 新增历史统计数据模型

新增：

```cpp
DailyUsageStat
DailyUsageStatsListModel
```

QML 通过 `AppViewModel::usageStats` 读取每日聚合记录。

模型角色包括：

- `date`
- `serviceId`
- `serviceName`
- `serviceType`
- `watchSeconds`
- `networkBytesIn`
- `networkBytesOut`
- `networkBytesTotal`

### 4. SQLite 聚合存储

新增表：

```sql
daily_usage_stats
```

主键：

```sql
(stat_date, service_id)
```

同一天同一服务的观看时长和流量会累加，不保存逐条播放明细，避免历史表快速膨胀。

保留策略：

- 初始化数据库时删除过期记录。
- 打开或刷新历史统计页时再次清理。
- 保留窗口为含今天在内的最近 30 天。

### 5. 观看时长统计

播放统计接入现有播放上报入口：

- `reportPlaybackStarted`
- `reportPlaybackProgress`
- `reportPlaybackStopped`

统计使用墙钟时间，并尊重暂停状态：

- 播放中累计。
- 暂停时不累计。
- 停止播放、切换页面清理播放状态、应用退出前会刷新 pending 数据。

为避免播放器状态频繁变化导致大量 SQLite 写入，观看时长先在内存中累积：

- 累计达到 15 秒后落库。
- 15 秒定时器兜底落库。
- 停止播放或退出应用前强制落库。

### 6. 网络流量统计

本次统计覆盖 Qt 可见网络路径：

- Emby / Jellyfin API 请求。
- WebDAV PROPFIND / MKCOL 请求。
- WebDAV 上传与下载任务。
- WebDAV 播放代理从远端读取的视频流字节。

网络流量同样先进入内存 pending buffer：

- 单服务累计达到 1 MB 后落库。
- 15 秒定时器兜底落库。
- 打开或刷新历史统计页前先刷新 pending 数据。

### 7. 播放器直连流量补充

初版历史统计只覆盖 Qt 网络层可见的网络路径，Emby、Jellyfin、IPTV 的正片视频流由 libmpv 直接拉取，不经过 `QNetworkAccessManager`。

后续已通过播放器层 mpv `cache-speed` 采样补齐直连视频输入流量，详见：

- `featDocs/2026-07-03-01.44.14.md`
- `bugFix/2026-07-03-01.39.43.md`

该补充仍然没有把播放链路改成 Qt 代理，也没有修改 mpv 播放 URL 流程，原因是：

- 避免影响 4K 在线播放性能。
- 避免重新引入历史播放器窗口、层级、`loadfile` 参数和 WebDAV Range 相关问题。
- WebDAV 继续由已有播放代理统计远端响应字节，避免把本机回环流量重复计算。
