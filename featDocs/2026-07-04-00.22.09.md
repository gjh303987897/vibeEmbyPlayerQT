# 缩略图加载动态提示

## 背景

媒体库、继续观看、详情页海报、演员头像和 IPTV 台标等图片使用异步加载。旧界面在图片未加载完成时只显示空的占位底色，用户无法判断图片是在加载中、没有图片，还是网络较慢。

本次增加缩略图加载动态提示：当缩略图仍处于加载状态时，在图块中央显示一个小型旋转点阵图标。

## 阅读与复查

实现前复查了以下内容：

- `qml/Main.qml`
  - 确认 `PosterImage` 是媒体库卡片、海报卡片、继续观看卡片、人物卡片、剧集卡片和详情页封面的统一缩略图组件。
  - 确认 IPTV 频道卡片与播放器侧边频道列表的台标没有复用 `PosterImage`，需要单独接入加载提示。
- 播放器相关历史修复文档
  - 确认本次只修改 QML 图片展示层，不修改 `PlayerController`、`MpvVideoItem`、native window 嵌入、播放控制层或 mpv 参数顺序。

## 实现内容

### 1. 新增 `ThumbnailLoadingIcon`

在 QML 中新增轻量组件 `ThumbnailLoadingIcon`：

- 使用半透明圆形底托，与深色/浅色主题协调。
- 使用 8 个小圆点组成旋转点阵。
- 通过 `RotationAnimation` 循环旋转。
- 通过 `running` 属性控制显示和动画。

### 2. `PosterImage` 自动显示加载提示

`PosterImage` 内部的 `Image` 增加 `id`，并根据图片状态显示加载图标：

```qml
running: imageUrl.length > 0 && posterImage.status === Image.Loading
```

因此所有复用 `PosterImage` 的缩略图都会自动获得加载反馈。

### 3. IPTV 台标加载提示

IPTV 页面频道卡片和播放器侧边频道列表的台标也接入 `ThumbnailLoadingIcon`。

播放器侧边频道列表原本未显式开启异步加载，本次同步设置：

```qml
asynchronous: true
```

避免频道列表小图加载影响界面响应。

## 影响范围

本次改动只影响：

- `qml/Main.qml` 中缩略图加载时的视觉反馈。
- IPTV 台标图片加载表现。

不影响：

- C++ 业务逻辑。
- 数据库存储。
- Emby/Jellyfin/WebDAV/SMB/IPTV 请求逻辑。
- 播放器窗口嵌入、播放、seek、字幕、音轨和控制层。
