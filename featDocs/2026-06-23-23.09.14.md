# 添加服务选择列表样式优化

## 背景

服务卡片页面点击添加后，弹出的添加服务对话框中服务类型选择列表仍然偏向原生控件观感，与当前应用的深色卡片、圆角输入框和高亮选中态不够一致。

本次优化目标是让服务选择列表更美观，并与现有 `ModernButton`、`ModernTextField`、播放器菜单和设置页选择控件保持一致。

## 阅读与复查

实现前复查了：

- `VIBEDOCS/MediaServices.md`
  - 确认添加服务流程由 QML 表单收集输入，保存仍交给 `AppViewModel::saveServiceCard()`。
  - QML 不参与服务登录、网络请求和持久化逻辑。
- `VIBEDOCS/SettingsAppearance.md`
  - 确认主题 token 应通过 `theme.bg`、`theme.elevated`、`theme.input`、`theme.text`、`theme.border`、`theme.primary` 等统一使用。
- `qml/Main.qml`
  - 确认添加服务对话框的服务类型选择使用 `ModernComboBox`。
  - 确认设置页主题、语言选择也复用 `ModernComboBox`，因此本次优化放在通用组件层，保持整体一致。

## 实现内容

### 1. 优化 `ModernComboBox` 主体

调整内容区右侧留白，为下拉箭头留出稳定空间。

新增自定义 indicator：

- 下拉关闭时显示向下箭头。
- 下拉打开时显示向上箭头。
- 使用 `theme.muted` / `theme.subtle`，与输入框视觉一致。

输入框边框在以下状态使用主题主色：

- 获得焦点。
- 下拉列表打开。

### 2. 优化下拉列表

新增自定义 `delegate`：

- 每个选项使用 38px 高度。
- 8px 圆角。
- hover 使用 `theme.elevatedHover`。
- 当前选中项使用 `theme.primary` 高亮。
- 当前选中项右侧显示圆点标记。
- 文本通过 `combo.textAt(index)` 获取，兼容添加服务弹窗的字符串列表和设置页的 `{ label, value }` 模型。

新增自定义 `popup`：

- 使用 10px 圆角。
- 背景使用 `theme.elevated`。
- 边框使用 `theme.primary` / `theme.border`。
- 内部列表带 6px padding。
- 最大高度限制为 252px，较长列表可滚动。

## 修改文件

- `qml/Main.qml`
  - 优化 `ModernComboBox` 的 indicator、delegate、popup 和背景样式。
