# 播放器缓冲加载提示

## 需求

当视频因为网络加载或缓冲短暂停住时，播放器不能只停在当前画面，否则用户容易误认为程序卡死。需要在播放暂停、进度条等控制之外增加一个美观的加载提示。

## 参考与避坑

实现前复查了 `/bugFix` 中的播放器相关记录：

- `bugFix/2026-06-21-15.11.08.md`
  - 之前修复过 mpv `loadfile` 参数顺序、播放开始上报过早、播放错误不上屏等问题。
- `bugFix/2026-06-21-15.32.01.md`
  - Windows 下播放器使用独立 native popup HWND，需要保持正确几何与 DPR 换算。
- `bugFix/2026-06-21-15.48.45.md`
  - 顶部/底部控制层曾经被 native mpv 视频窗口盖住，后续通过独立透明工具窗口和 `raiseChromeWindows()` 修复。
- `bugFix/2026-06-22-14.26.39.md`
  - WebDAV 自签名证书播放曾经依赖 `tls-verify` 状态。
- `bugFix/2026-06-22-14.35.14.md`
  - WebDAV 播放后来改为本地代理，避免 libmpv 直接访问远端 WebDAV 时认证失败。

本次没有修改播放 URL、`loadfile` 参数、WebDAV 本地播放代理、TLS 或 HTTP 认证逻辑，只新增 mpv 状态观察与 UI 提示。

## 实现内容

### C++ 播放状态

- `PlayerController` 新增只读状态：
  - `loading`
  - `buffering`
  - `bufferingProgress`
  - `seeking`
- 通过 mpv 官方属性观察状态：
  - `paused-for-cache`
  - `cache-buffering-state`
  - `seeking`
  - `idle-active`
- 播放请求发出后先进入 `loading=true`。
- `MPV_EVENT_FILE_LOADED`、`MPV_EVENT_PLAYBACK_RESTART`、播放结束或 idle 时清理加载/缓冲状态。
- `loadfile` 命令如果同步下发失败，会立即清理 `loading` 状态，避免加载提示残留。

### QML 播放器提示层

- `MpvVideoItem` 转发上述播放状态给 QML。
- `PlayerPage` 增加 300ms 延迟显示，避免正常短暂加载或 seek 时闪烁。
- 新增 `playerLoadingWindow` 透明工具窗口：
  - 覆盖播放器可视区域。
  - 居中显示 `BusyIndicator`、状态标题和辅助提示。
  - 支持显示缓冲百分比。
  - 使用 `Qt.WindowTransparentForInput`，尽量不拦截鼠标输入。
- 将加载提示窗口纳入 `raiseChromeWindows()`，并在顶部/底部控制窗口显示时同步抬升，避免再次被 Windows native mpv popup HWND 盖住。

### 文案

新增翻译键：

- `player.loading`
- `player.buffering`
- `player.seeking`
- `player.networkHint`

## 影响范围

- 影响播放器 UI 与 mpv 状态观察。
- 不影响 Emby/Jellyfin/WebDAV/IPTV 的播放 URL 生成。
- 不影响 WebDAV 本地播放代理、证书处理或认证转发。
- 不改变现有播放暂停、进度条、快进、字幕、音轨、倍速、音量逻辑。

## 验证重点

- 本地视频正常播放。
- 初始加载较慢时能看到加载提示。
- 网络缓冲时能看到缓冲提示。
- seek 较慢时能看到定位提示。
- 控制层和加载提示不会被 mpv native 视频窗口盖住。
- 播放结束或退出播放后提示不会残留。
