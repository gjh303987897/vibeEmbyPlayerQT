# Linux AppImage 打包支持

## 背景

项目已有 GitHub Actions 跨平台发布流程，Linux 平台会产出 `tar.gz` 安装包。

为了让 Linux 用户可以直接下载单文件运行，本次为 Linux x86_64 与 Linux arm64 增加 AppImage 打包产物，同时保留原有 `tar.gz` 包。

## 阅读与复查

实现前复查了以下内容：

- `.github/workflows/build-release.yml`
  - 确认现有 Linux 构建矩阵包含 `linux-x86_64` 与 `linux-arm64`。
  - 确认 Linux 安装目录先由 `cmake --install` 生成，再从同一安装目录打包 `tar.gz`。
  - 确认 Release 发布阶段会对上传资产做后置检查。
- `CMakeLists.txt`
  - 确认 Linux 已安装 desktop 文件和 hicolor 应用图标。
  - 确认本次无需调整播放器、资源加载或安装目标。
- `packaging/linux/vibePlayerQT.desktop.in`
  - 确认 desktop 文件已包含 `Name`、`Exec`、`Icon`、`Categories` 等 AppImage 根 desktop 文件需要的基础字段。
- AppImage 官方 `appimagetool`
  - 使用 `AppImage/appimagetool` continuous release 提供的 `appimagetool-x86_64.AppImage` 与 `appimagetool-aarch64.AppImage`。
  - CI 运行工具时设置 `APPIMAGE_EXTRACT_AND_RUN=1`，避免 GitHub runner 对 FUSE 的依赖。

本次改动不涉及播放器控制、libmpv 调用、native window 嵌入、WebDAV 播放代理、Emby/Jellyfin API 或历史统计逻辑。

## 实现内容

### 1. Linux CI 额外生成 AppImage

在 Linux `tar.gz` 打包步骤后新增 `Package Linux AppImage`。

流程如下：

- 根据矩阵包名映射 AppImage 架构：
  - `linux-x86_64` -> `x86_64`
  - `linux-arm64` -> `aarch64`
- 将 `$PACKAGE_DIR` 复制为临时 `vibePlayerQT.AppDir`。
- 在 AppDir 根目录生成 `AppRun`：
  - 设置 `LD_LIBRARY_PATH="$app_dir/lib"`。
  - 启动 `bin/vibePlayerQT`。
- 将安装目录中的 desktop 文件复制到 AppDir 根目录，并把 `Exec` 改为 `AppRun`。
- 将安装目录中的应用图标复制到 AppDir 根目录为 `vibePlayerQT.png`，并创建 `.DirIcon`。
- 下载对应架构的 `appimagetool`。
- 输出：

```text
dist/vibePlayerQT-${PACKAGE_VERSION}-linux-x86_64.AppImage
dist/vibePlayerQT-${PACKAGE_VERSION}-linux-arm64.AppImage
```

### 2. 保留原有 tar.gz 包

原有 Linux `tar.gz` 打包步骤没有删除或改写。

Release 中 Linux 平台现在同时包含：

- `tar.gz`
- `AppImage`

这样已有用户和自动化脚本仍可继续使用压缩包，新用户可以选择单文件 AppImage。

### 3. Release 资产检查支持 AppImage

发布阶段的 `Guard release assets` 增加 `.AppImage` 分支。

由于 Release job 运行在 x86_64 Ubuntu 上，不能可靠执行或展开 arm64 AppImage，因此这里对 `.AppImage` 做文件类型检查；内容来源仍然是同一份已检查过的 Linux 安装目录和 `tar.gz` 打包输入。

### 4. README 更新

同步更新中英文 README 的 GitHub Releases 说明：

- Linux x86_64 tar.gz and AppImage
- Linux arm64 tar.gz and AppImage

## 影响范围

本次改动只影响：

- GitHub Actions 发布打包流程。
- README 发布产物说明。
- 本功能文档。

不影响：

- C++ 源码。
- QML 界面。
- SQLite 数据。
- 播放器初始化、播放、暂停、seek、字幕、音轨。
- Emby/Jellyfin/WebDAV/SMB/IPTV 服务逻辑。

- 检查 AppDir 根目录所需 `AppRun`、desktop 文件、图标文件生成逻辑。
