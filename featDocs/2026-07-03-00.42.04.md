# 隐私模式

## 背景

用户需要在服务页顶部增加一个锁图标按钮，用于进入隐私模式。隐私模式下：

- 服务页显示被标记为隐私的服务卡片。
- 顶部额外显示 `隐私卡片编辑` 按钮。
- 用户可以选择哪些服务卡片只在隐私模式中显示。
- 进入隐私模式需要 PIN。
- PIN 可在设置页设置或修改，修改前必须输入旧 PIN。
- 普通模式历史统计不包含隐私模式记录。
- 隐私模式下历史统计包含隐私记录，并对隐私记录做标注。

## 实现内容

### 1. 服务卡片隐私标记

在 `servers` 表新增：

```sql
private_mode INTEGER NOT NULL DEFAULT 0
```

普通模式加载：

```sql
enabled = 1 AND private_mode = 0
```

隐私模式加载：

```sql
enabled = 1 AND private_mode = 1
```

没有复用 `enabled` 字段，避免影响已有软删除、删除本地数据、排序和会话恢复逻辑。

### 2. 顶部锁图标入口

服务页顶部新增锁图标按钮，位于 `历史统计`、`添加`、`编辑`、`设置` 前。

- 未进入隐私模式时显示锁图标。
- 已进入隐私模式时显示解锁图标。
- 未设置 PIN 时点击会跳转到设置页，并提示先设置 PIN。
- 已设置 PIN 时点击会弹出 PIN 输入框。
- 隐私模式下点击会退出隐私模式，并回到普通服务页。

### 3. 隐私卡片编辑

隐私模式下顶部新增 `隐私卡片编辑` 按钮。

点击后弹出编辑对话框：

- 列出全部启用服务卡片。
- 每行使用复选框控制该服务是否属于隐私卡片。
- 勾选后该卡片只在隐私模式中显示。
- 取消勾选后该卡片回到普通模式显示。

新增 `ServiceCardListModel::privateMode` 角色供 QML 使用。

### 4. PIN 设置与校验

设置页新增 `隐私` 分组。

首次设置 PIN：

- 输入新 PIN。
- 再次确认新 PIN。

修改 PIN：

- 必须输入旧 PIN。
- 输入新 PIN。
- 再次确认新 PIN。

PIN 规则：

- 4 到 12 位数字。
- 不保存明文 PIN。
- 使用随机 salt 和 SHA-256 hash 存储到 `QSettings`。

### 5. 历史统计隐私隔离

历史统计表新增：

```sql
privacy_mode INTEGER NOT NULL DEFAULT 0
```

主键从：

```sql
(stat_date, service_id)
```

迁移为：

```sql
(stat_date, service_id, privacy_mode)
```

这样同一天同一服务的普通记录和隐私记录不会互相覆盖。

兼容迁移策略：

- 旧表没有 `privacy_mode` 时，原有记录迁移为普通模式记录。
- 如果表已有 `privacy_mode` 但不在主键中，也会重建表并迁移。

### 6. 历史统计显示规则

普通模式：

- 只加载 `privacy_mode = 0` 的记录。
- 总观看时长和总流量不包含隐私记录。

隐私模式：

- 加载普通记录和隐私记录。
- 隐私记录显示 `隐私模式 / Private` 徽标。
- 总观看时长和总流量会包含隐私模式记录。

### 7. 播放与流量统计

观看时长与流量 pending buffer 现在按：

```text
service_id + privacy_mode
```

分开累计，避免落库前混合普通和隐私数据。

WebDAV 流量信号会在 `AppViewModel` 中补全当前 WebDAV 卡片的隐私标记，确保隐私 WebDAV 播放代理、下载、上传和目录请求都能正确归类。
