# 播放器视频信息检测

## 需求

在播放器已有播放/暂停、进度条、字幕、音轨、倍速和音量控制之外，增加美观的视频信息检测展示，包含：

- 分辨率
- 编码格式
- 帧率
- 码率

同时需要特别避免复发历史播放器问题，尤其是原生 mpv 窗口层级、`loadfile` 参数、WebDAV 播放代理、TLS 与认证处理相关问题。

- `bugFix/2026-06-21-15.32.01.md`
  - 重点确认 Windows 原生 mpv popup HWND 的几何与 DPR 换算不受影响。

本次还参考了 mpv 官方手册中 `track-list` 属性的节点结构，使用当前视频轨道的 `codec`、`demux-w`、`demux-h`、`demux-fps`、`demux-bitrate` 等字段。

## C++ 实现

### `PlayerController`

新增只读属性：

- `videoResolution`
- `videoCodec`
- `videoFrameRate`
- `videoBitrate`

新增信号：

- `videoInfoChanged()`

实现方式：

- 继续复用已有 `track-list` 观察与 `updateTracks()` 流程。
- 从 `track-list` 中筛选 `type == "video"` 的轨道。
- 优先使用 `selected == true` 的视频轨道；如果 mpv 尚未标记选中轨道，则使用第一个视频轨道。
- 读取并格式化：
  - `demux-w` + `demux-h` -> `1920 x 1080`
  - `codec` -> 大写编码名，例如 `H264` / `HEVC`
  - `demux-fps` -> `23.98 fps`
  - `demux-bitrate` / `hls-bitrate` -> `8.5 Mbps`
- 播放新 URL 前先清空旧视频信息，避免短时间显示上一条视频的残留信息。
- `FILE_LOADED`、`VIDEO_RECONFIG`、`PLAYBACK_RESTART` 时主动刷新 `track-list`，让信息尽快出现在 UI 上。
- `stop()` / `shutdown()` 清理播放状态时同步清空视频信息。

### `MpvVideoItem`

新增 QML 可绑定属性：

- `videoResolution`
- `videoCodec`
- `videoFrameRate`
- `videoBitrate`

并转发 `PlayerController::videoInfoChanged()` 给 QML。

## QML 实现

### 播放器信息按钮

在播放器底部控制条中新增 `视频信息 / Info` 按钮：

- 点击后切换视频信息面板显示状态。
- 切换后调用 `revealControls()` 与 `raiseChromeWindows()`，保证浮层在 native 视频窗口之上。

### 视频信息面板

新增 `playerInfoWindow`：

- 使用 `Qt.FramelessWindowHint | Qt.Tool` 的独立工具窗口。
- 放置在播放器右上角，不覆盖底部控制条。
- 暗色半透明背景、浅色文字、细边框。
- 展示四项信息：分辨率、编码格式、帧率、码率。
- 如果某项暂时无法从 mpv 获得，显示 `--`。
- 提供暗色关闭按钮。
- `Esc` 键可关闭信息面板；其他播放器快捷键继续转交给播放器页面处理。
- 离开播放器页面或确认退出播放时自动隐藏信息面板。

## 避坑说明

本次没有修改：

- `PlayerController::playUrl()` 中的 `loadfile` 参数顺序。
- Emby / Jellyfin 播放 URL 生成。
- WebDAV 本地播放代理。
- WebDAV Basic / Digest 认证转发。
- WebDAV 自签名证书处理。
- Windows 原生 mpv popup HWND 创建、几何换算和 Z 序逻辑。
- 现有播放/暂停、进度条、字幕、音轨、倍速和音量控制逻辑。

新增面板采用独立工具窗口，并纳入 `raiseChromeWindows()`，避免复发控制层被原生视频窗口遮挡的问题。

## 后续验证重点

- 本地视频播放时信息面板能显示分辨率、编码格式、帧率和码率。
- 无码率信息的媒体显示 `--`，不影响播放。
- 播放器控制层、加载提示和视频信息面板都不会被 native mpv 窗口遮挡。
- WebDAV 播放仍通过本地代理正常播放。
- 切换视频或退出播放后不会残留上一条视频信息。
