# 播放器字幕与音轨选择列表样式优化

## 需求

优化播放器中“字幕”和“音轨”按钮点击后弹出的选择列表样式，使其更美观，并与播放器已有的深色控制条、视频信息面板、缓存提示等组件更加一致。

同时需要避免影响播放器此前修复过的敏感路径，包括：

- 不修改 mpv `loadfile` 参数顺序。
- 不修改 Emby / Jellyfin / WebDAV / IPTV 的播放 URL 生成。
- 不修改 WebDAV 本地播放代理、认证转发、Range 请求或 TLS 处理。
- 不修改 Windows native mpv popup HWND 创建、DPR 换算和 Z 序逻辑。
- 不影响播放器已有的播放/暂停、进度条、缓冲提示、视频信息与缓存统计逻辑。

  - 确认最近的 seek 缓冲提示依赖 mpv `PLAYBACK_RESTART` 事件，本次不修改这条状态机。

## 实现内容

### 1. 替换字幕/音轨原生菜单

原实现使用 Qt Quick Controls 的 `Menu` / `MenuItem`：

- 字幕按钮内嵌 `subtitlesMenu`。
- 音轨按钮内嵌 `audioMenu`。
- 原生菜单视觉与播放器深色控制条不够统一。

本次改为播放器内自定义轨道选择浮层：

- 新增 `playerTrackMenuWindow` 独立透明工具窗口。
- 使用深色半透明面板、细边框、圆角列表项和选中状态标记。
- 字幕和音轨共用同一个浮层，通过 `trackMenuMode` 切换内容。
- 字幕列表顶部保留“关闭”条目。
- 真实轨道列表直接绑定 `mpvVideo.subtitleTracks` / `mpvVideo.audioTracks`，继续复用 C++ 暴露的 `TrackListModel`。

### 2. 浮层定位与层级

新增播放器状态：

- `trackMenuVisible`
- `trackMenuMode`
- `trackMenuAnchorItem`
- `trackMenuAnchorGlobalX`

点击字幕或音轨按钮时：

- 记录按钮中心的全局坐标。
- 将浮层定位在底部控制条上方，并尽量对齐触发按钮。
- 自动限制浮层左右边界，避免超出播放器区域。
- 通过 `raiseChromeWindows()` 统一抬升，避免被 Windows native mpv popup HWND 遮挡。

### 3. 交互细节

- 打开轨道浮层时停止控制条自动隐藏，避免用户正在选择时控制条消失。
- `Esc` 可以关闭浮层。
- 点击视频区域会关闭浮层并重新显示控制条。
- 选择字幕或音轨后自动关闭浮层。
- 轨道项过长时使用省略号，避免文本溢出。
- 选中轨道使用蓝色圆形勾选状态，与现有播放器高亮色保持一致。
- 轨道编码显示为右侧短标签，提升可读性。

### 4. 翻译

新增翻译键：

- 英文：`player.tracks` -> `tracks`
- 中文：`player.tracks` -> `条轨道`

用于显示列表数量，例如：

- `2 tracks`
- `2 条轨道`
