# 继续观看详情页季集支持实现记录

## 需求

上一轮已在媒体库中的 Series 详情页增加季/集支持。本次补充要求：从“继续观看”进入的视频简介页面也要显示对应的剧集/季信息。

典型场景：

- 继续观看列表中是一集电视剧或纪录片 Episode。
- 用户点击该 Episode 进入详情页。
- 详情页除了保留该集的简介、播放按钮和进度，也展示它所属 Series 的季列表和当前季的集列表。

## 官方 API 依据

本次未新增 API 端点，继续复用上一轮已确认的官方接口：

- Emby REST API
  - `GET /Shows/{Id}/Seasons`
  - `GET /Shows/{Id}/Episodes`
  - Episodes 支持 `SeasonId`、`UserId`、`Fields`
- Jellyfin OpenAPI
  - `GET /Shows/{seriesId}/Seasons`
  - `GET /Shows/{seriesId}/Episodes`
  - Episodes 支持 `seasonId`、`userId`、`fields`

本次额外确认了 Emby `Fields` 可包含 `ParentId`，Jellyfin 对应字段也可通过 `fields=ParentId` 请求。这样 Episode 详情可用 `ParentId` 对应 season id，高亮当前集所在季。

## 代码改动

### ViewModel 判断逻辑

文件：

- `src/viewmodels/AppViewModel.h`
- `src/viewmodels/AppViewModel.cpp`

新增属性：

- `selectedItemHasSeriesEpisodes`

语义：

- `Series`：可以显示季/集区域。
- `Episode` 且带 `seriesId`：也可以显示季/集区域。
- `Movie` 或其它无 Series 上下文的条目：不显示季/集区域。

保留原有 `selectedItemIsSeries`：

- 只用于判断当前详情是否是 Series 本身。
- Series 本身仍隐藏直接播放按钮。
- Episode 详情仍保留播放/继续播放按钮。

### 继续观看入口加载季集

文件：

- `src/viewmodels/AppViewModel.cpp`

调整：

- `openContinueItem()` 获取 Episode 详情成功后，不再只判断 `selectedItemIsSeries()`。
- 改为判断 `selectedItemHasSeriesEpisodes()`。
- 如果当前详情是继续观看的 Episode，并且服务端返回 `SeriesId`，就自动调用 `loadSeriesSeasons()`。

### 默认选中当前 Episode 所在季

文件：

- `src/viewmodels/AppViewModel.cpp`

调整：

- `loadSeriesSeasons()` 支持从当前 Episode 提取：
  - `parentId`：优先匹配 season id。
  - `parentIndexNumber`：作为 season number 兜底匹配。
- seasons 返回后优先选中当前 Episode 所在季。
- 如果无法匹配，则回退选中第一季。

### 请求字段补充

文件：

- `src/services/emby/EmbyClient.cpp`
- `src/services/jellyfin/JellyfinClient.cpp`

调整：

- 继续观看列表请求字段补充 `ParentId`。
- Episode 列表请求字段补充 `ParentId`。
- Item detail 请求字段补充 `ParentId`。

目的：

- 确保 Episode 详情能够识别所属 season。
- 即使部分服务端响应不带 `ParentId`，仍可以用 `ParentIndexNumber` 兜底。

### QML 展示条件

文件：

- `qml/Main.qml`

调整：

- 季/集区域从 `selectedItemIsSeries` 改为 `selectedItemHasSeriesEpisodes`。
- Series 详情页显示季/集区域。
- 从继续观看进入的 Episode 详情页也显示季/集区域。
- Episode 的播放按钮仍显示；Series 的直接播放按钮仍隐藏。
