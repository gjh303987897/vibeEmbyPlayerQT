## 我修复的问题

### 1. 修复 mpv `invalid parameter`

问题：

- 播放时出现 `invalid parameter`。
- 带断点续播 `start=...` 时更容易触发。

原因：

- 当前 mpv 的 `loadfile` 第 3 个参数是 `index`。
- 原代码把 `start=...` 直接放在第 3 个参数位置，mpv 会把它当成错误的 index。

修改：

- 在 `src/player/PlayerController.cpp` 里把命令参数从：

```text
loadfile <url> replace start=<seconds>
```

改为：

```text
loadfile <url> replace -1 start=<seconds>
```

结果：

- `invalid parameter` 消失。
- 进度条开始正常走动。

### 2. 修复 Emby/Jellyfin 播放 URL 生成流程

问题：

- 原来直接拼 `/Videos/{id}/stream?static=true&api_key=...`。
- 这个流程缺少播放会话信息。

修改：

- 把 `MediaServiceClient` 的播放 URL 获取从同步接口改为异步接口 `fetchPlaybackUrl(...)`。
- 在 `MediaServerClientBase` 里新增 `fetchPlaybackUrlWithScheme(...)`。
- 播放前先请求：

```text
/Items/{id}/PlaybackInfo
```

- 从返回结果解析：

```text
MediaSources[0].Id
PlaySessionId
```

- 再生成带 `MediaSourceId` 和 `PlaySessionId` 的 stream URL。
- 播放上报 `Sessions/Playing`、`Sessions/Playing/Progress`、`Sessions/Playing/Stopped` 时也带上 `MediaSourceId` 和 `PlaySessionId`。

### 3. 修复播放错误没有显示到界面

问题：

- `MpvVideoItem::errorOccurred` 原来只在 QML 里 `console.warn`。
- 用户界面错误栏看不到真实播放器错误。

修改：

- 在 `AppViewModel` 新增 `reportPlaybackError(...)`。
- 在 `qml/Main.qml` 中把 `onErrorOccurred` 改为调用：

```qml
appViewModel.reportPlaybackError(message)
```

### 4. 增加 mpv 日志和结束错误记录

问题：

- 之前只能看到黑屏或 `00:00`，无法确认 mpv 是否真的加载、解码、输出。

修改：

- `PlayerController` 初始化时调用：

```cpp
mpv_request_log_messages(m_mpv, "info");
```

- 处理 `MPV_EVENT_LOG_MESSAGE` 并写入应用日志。
- 处理 `MPV_EVENT_END_FILE`，如果 reason 是 error，就记录 `mpv_error_string(...)` 并上报 UI。

验证时日志确认 mpv 已经输出：

```text
VO: [gpu-next] 640x360 yuv420p
```

### 5. 修复播放开始上报过早

问题：

- QML 原来在 `sourceChanged` 和 `Component.onCompleted` 时就调用 `reportPlaybackStarted()`。
- 这会导致媒体还没真正加载成功时就上报播放开始。

修改：

- 改成只有 `duration > 0` 或 `position > 0` 后才调用 `reportPlaybackStarted()`。

### 6. 修复 Qt Debug `UniqueConnection` 断言

问题：

- 每次测试启动时弹出断言：

```text
QObject::connect: Unique connection requires the slot to be a pointer to a member function of a QObject subclass.
```

原因：

- `MpvVideoItem` 中对 lambda connect 使用了 `Qt::UniqueConnection`。
- Qt 不允许 lambda/functor slot 使用 `Qt::UniqueConnection`。

修改：

- 移除了该 lambda connect 上的 `Qt::UniqueConnection`。

结果：

- 测试启动时不再出现这个断言。

## 我为黑屏做过的修改和验证

### 1. 调整 native window 刷新

修改：

- `PlayerController` 在 `MPV_EVENT_FILE_LOADED`、`MPV_EVENT_VIDEO_RECONFIG`、`MPV_EVENT_PLAYBACK_RESTART` 时发出 `videoOutputChanged()`。
- `MpvVideoItem` 收到后重新同步、显示并抬高 native window。
- 播放请求后增加延迟刷新。

结果：

- mpv 日志显示正常加载和输出视频。
- 用户反馈进度条正常，但画面仍黑屏。

### 2. 改 Windows 原生 HWND 承载

修改：

- 在 Windows 下注册 `VibePlayerMpvHostWindow`。
- 创建原生 Win32 HWND 给 mpv 作为 `wid`。
- 处理 `WM_ERASEBKGND`，避免背景擦除。
- 处理 `WM_NCHITTEST`，避免视频窗口抢输入。
- 给父窗口加 `WS_CLIPCHILDREN | WS_CLIPSIBLINGS`。

结果：

- 日志显示 libmpv 初始化成功。
- 日志显示 `VO: [gpu-next] 640x360 yuv420p`。
- 用户反馈画面仍黑屏。

### 3. 尝试独立 popup 视频窗口

修改：

- 把视频承载窗口从 child HWND 改成独立 `WS_POPUP` 工具窗口。
- 使用全局坐标跟随播放器区域。
- 避免同步 `RedrawWindow(...UPDATENOW)`。

结果：

- 日志显示 popup HWND 创建成功。
- 日志显示 libmpv 初始化成功。
- 日志显示 mpv 正常输出视频。

### 4. 移除 Qt Quick alpha buffer

修改：

- 从 `src/app/main.cpp` 移除了：

```cpp
QQuickWindow::setDefaultAlphaBuffer(true);
```

原因：

- 播放器使用 native window embedding，透明 alpha buffer 可能影响 Windows 下 Qt Quick 与原生窗口的合成。

### 5. 播放页不再绘制黑色背景

修改：

- 把 `qml/Main.qml` 里的 `PlayerPage` 从 `Rectangle` 改为 `Item`。
- 删除播放器页自身的黑色背景绘制。

原因：

- 避免 QML 播放页背景持续覆盖或擦黑原生视频窗口。
