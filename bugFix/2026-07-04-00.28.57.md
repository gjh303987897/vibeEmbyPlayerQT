# 修复媒体库返回按钮直接回到服务卡片页

## 问题现象

在 Emby 服务中：

1. 从服务卡片进入 Emby 主页面。
2. 点击某个媒体库进入媒体库内容页。
3. 点击左上角返回按钮。

旧行为会直接返回服务卡片页面。

正确行为应该是返回上一级 Emby 主页面，也就是包含“继续观看”和“媒体库”的服务首页。

由于 Emby 和 Jellyfin 共用 `AppViewModel` 中的媒体库导航逻辑，该问题同样可能影响 Jellyfin。

## 原因分析

页面结构为：

```text
services -> home -> library -> details/player
```

QML 顶部返回按钮在 `currentView === "library"` 时会调用：

```cpp
AppViewModel::mediaLibraryBack()
```

该函数需要区分两种情况：

- 当前位于媒体库内的子文件夹：返回上一级媒体目录。
- 当前位于媒体库根目录：返回服务首页 `home`。

旧实现中，当 `m_mediaParentHistory` 为空时调用了 `backToServices()`，导致清空当前会话、媒体库、继续观看和列表数据，然后回到服务卡片页。

## 修复内容

将 `mediaLibraryBack()` 在没有媒体库父级历史时的返回目标改为 `backToHome()`：

```cpp
if (m_mediaParentHistory.empty()) {
    backToHome();
    return;
}
```

同时在异常状态 `!m_currentLibrary` 时也使用 `backToHome()`，让有有效会话时回到服务首页；没有会话时 `backToHome()` 仍会回到服务卡片页。

## 没有改动的内容

本次没有修改：

- Emby API 路径。
- Jellyfin API 路径。
- 媒体库、媒体项、详情页的网络请求参数。
- `PlayerController`。
- `MpvVideoItem`。
- mpv `loadfile` 参数。
- native window、DPR、geometry 和 Z 序逻辑。
- WebDAV / SMB / IPTV 逻辑。


本地构建通过。

需要人工点测：

- Emby 主页面 -> 媒体库 -> 返回：回到 Emby 主页面。
- Jellyfin 主页面 -> 媒体库 -> 返回：回到 Jellyfin 主页面。
- 媒体库内子文件夹 -> 返回：仍回到上一级媒体目录。
- 服务首页 -> 返回：仍回到服务卡片页面。
