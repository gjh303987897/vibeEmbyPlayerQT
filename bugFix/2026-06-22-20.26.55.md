# 修复 seek 到未缓存区域时缓冲提示不显示

## 问题现象

用户正在播放视频时，如果当前只缓存了未来约 25 秒，但用户拖动进度条到未来 120 秒这种明显超出缓存范围的位置，播放器可能不会显示“正在缓冲 / 正在加载”的提示。

结果是画面停住时用户容易误以为播放器卡死。

## 复查过的历史记录

修复前重新阅读了播放器相关记录：

- `bugFix/2026-06-21-15.11.08.md`
  - 重点确认不改动 `loadfile` 参数顺序。
  - 不改动播放开始上报条件。
- `bugFix/2026-06-21-15.48.45.md`
  - 重点确认播放器提示层必须继续通过独立工具窗口和 `raiseChromeWindows()` 抬升，避免被 Windows native mpv popup HWND 遮挡。
- `bugFix/2026-06-22-14.35.14.md`
  - 重点确认不改动 WebDAV 本地播放代理、认证转发和 Range 请求处理。
- `featDocs/2026-06-22-16.54.53.md`
  - 复查了播放器缓冲加载提示最初实现。

同时查询了 mpv 官方手册中 `seeking`、`paused-for-cache`、`cache-buffering-state` 和 `playback-restart` 的含义。`seeking` 表示 mpv 正在 seek 或尝试重启播放，但实际在某些情况下这个状态可能很短。

## 原因分析

旧逻辑中，播放器加载提示依赖：

- `mpvVideo.loading`
- `mpvVideo.buffering`
- `mpvVideo.seeking`

并且为了避免普通短暂加载闪烁，QML 里有一个 300ms 延迟显示：

- 状态开始后 300ms 才真正显示 `playerLoadingWindow`
- 如果 300ms 内 `seeking` 变回 `false`，延迟定时器会被取消

当用户主动拖动到未缓存区域时，mpv 的 `seeking` 可能很快结束，但网络数据还没真正追上。于是出现：

1. 用户 seek 到未缓存位置。
2. `seeking` 短暂变为 `true`。
3. 300ms 延迟尚未触发时 `seeking` 又变为 `false`。
4. 加载提示被取消。
5. 视频仍在等待新位置的数据，界面却没有提示。

## 修复内容

### QML 播放页状态机

在 `PlayerPage` 增加：

- `seekLoadingActive`
- `beginSeekLoading()`
- `finishSeekLoading()`
- `seekLoadingTimeout`

新的逻辑：

- 用户主动 seek 时调用 `beginSeekLoading()`。
- `rawPlaybackLoading` 现在包含 `seekLoadingActive`。
- 300ms 后如果播放还没恢复，会显示已有的 `playerLoadingWindow`。
- 标题优先显示“正在缓冲”，更符合 seek 到未缓存区域时的用户感知。
- 收到 mpv 的 `PLAYBACK_RESTART` 后调用 `finishSeekLoading()`，关闭主动 seek 等待状态。
- 15 秒超时兜底，避免极端情况下状态残留。
- 离开播放器页面时清理 `seekLoadingActive` 和超时定时器。

覆盖的 seek 入口：

- 进度条拖动 `seekAbsolute`
- 左右方向键 `seekRelative`
- 底部控制条 `-15 秒`
- 底部控制条 `+15 秒`

### C++ 播放恢复信号

在 `PlayerController` 中新增信号：

- `playbackRestarted()`

当 mpv 发出 `MPV_EVENT_PLAYBACK_RESTART` 时触发该信号。

在 `MpvVideoItem` 中转发该信号给 QML：

- `onPlaybackRestarted: playerPage.finishSeekLoading()`

这样 QML 不再依赖 native 窗口刷新事件来判断播放恢复，而是直接使用 mpv 的播放恢复事件。

## 没有改动的内容

本次没有修改：

- `loadfile` 参数。
- Emby / Jellyfin 播放 URL。
- WebDAV 本地播放代理。
- WebDAV Basic / Digest 认证转发。
- WebDAV 自签名证书处理。
- Windows native mpv popup HWND 创建、DPR 换算和 Z 序逻辑。
- 已有播放器加载提示窗口的层级方案。
