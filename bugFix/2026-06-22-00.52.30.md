# IPTV 运行检查与启动问题修复回顾

## 背景

在完成 IPTV 第一版功能后，执行了一次本地运行检查。构建已经通过，但实际启动 `build-clang/vibePlayerQT.exe` 后程序很快退出，没有稳定显示主窗口。

## 当时做了什么

1. 启动构建产物：
   - 运行 `build-clang/vibePlayerQT.exe`。
   - 首次启动进程出现后很快退出。

2. 检查进程状态：
   - 发现进程不再存在。
   - 使用等待模式再次启动，读取到退出码 `-1`。

3. 判断问题范围：
   - 退出码 `-1` 对应 `src/app/main.cpp` 中 `QQmlApplicationEngine::objectCreationFailed` 的退出路径。
   - 初步判断不是 C++ 编译问题，而是 QML 根对象创建失败。

4. 定位 QML 具体错误：
   - 新增 `QQmlApplicationEngine::warnings` 日志输出。
   - 通过 `VIBEPLAYER_LOG_FILE` 写入启动日志。
   - 启动后日志显示：
     - `QtQuick.Dialogs` 模块未安装。
     - QML root object creation failed。

## 修复了哪些 Bug

### 1. 修复启动后立即退出

问题原因：

- IPTV 添加表单中新增了 `import QtQuick.Dialogs` 和 `FileDialog`。
- 运行目录中没有可用的 `QtQuick.Dialogs` QML 模块。
- QML 根对象创建失败，应用直接退出。

修复方式：

- 移除 `QtQuick.Dialogs` 依赖。
- 移除 QML `FileDialog`。
- IPTV 第一版改为在添加服务弹窗中输入或粘贴本地 `.m3u` / `.m3u8` 文件路径。
- 移除 CMake 中临时加入的 `QuickDialogs2` 依赖，避免引入未部署完整的 QML 模块。

结果：

- 应用可以正常启动。
- 主窗口可以保持运行。
- 进程状态 `Responding=True`。

### 2. 增加 QML 启动期 warning 日志

问题原因：

- 原先 QML 根对象创建失败时只返回 `-1`，缺少具体错误信息。
- 排查启动失败时不够直接。

修复方式：

- 在 `src/app/main.cpp` 中连接 `QQmlApplicationEngine::warnings`。
- 使用 `AppLogger::warning("qml", ...)` 记录 QML warning。
- 在 `objectCreationFailed` 时记录 `QML root object creation failed`。

结果：

- 后续如果 QML 模块缺失、绑定错误或语法加载失败，可以通过 `VIBEPLAYER_LOG_FILE` 快速看到启动期错误。

## 修复后又遇到了什么

修复 `QtQuick.Dialogs` 依赖后再次构建并运行：

- 构建通过。
- 程序成功启动。
- 检查进程状态时看到：
  - 进程存在。
  - `Responding=True`。

因此没有继续遇到新的启动阻断问题。

## 最终验证

- 重新执行 `.\scripts\build-clang.cmd`，构建通过。
- 启动 `build-clang/vibePlayerQT.exe`。
- 运行 3 秒后进程仍存在。
- 程序窗口可用于查看服务页和 IPTV 添加入口。
