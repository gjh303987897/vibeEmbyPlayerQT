# 修复详情页返回按钮直接回到服务首页

## 问题现象

在 Emby 服务中：

1. 从 Emby 主页面进入某个媒体库。
2. 在媒体库中选择一部影片进入详情页。
3. 点击左上角返回按钮。

旧行为会直接返回 Emby 主页面。

正确行为应该是返回上一级媒体库列表。

由于 Emby 和 Jellyfin 共用 `AppViewModel` 中的媒体导航逻辑，该问题同样可能影响 Jellyfin。

## 原因分析

页面结构为：

```text
services -> home -> library -> details/player
```

QML 顶部返回按钮此前只对 `library` 页面做了专门处理：

```qml
appViewModel.mediaLibraryBack()
```

当当前页面是 `details` 时，会落到通用返回逻辑：

```qml
appViewModel.backToHome()
```

而 `backToHome()` 会清空当前媒体库、媒体目录状态、详情项和媒体库条目列表，然后回到服务首页。因此从媒体库条目进入详情页后，返回按钮无法回到原媒体库列表。

## 修复内容

### 1. 新增详情页返回入口

`AppViewModel` 新增：

```cpp
Q_INVOKABLE void mediaDetailsBack();
```

当仍有 `m_currentLibrary` 时，说明详情页来自媒体库上下文。此时只清理详情页相关状态：

- `m_selectedItem`
- series seasons / episodes
- selected people
- 当前播放上下文

然后回到：

```cpp
setCurrentView(QStringLiteral("library"));
```

媒体库列表 `m_items`、当前媒体库 `m_currentLibrary` 和媒体目录位置不会被清掉。

### 2. QML 顶部返回按钮区分详情页

顶部返回按钮新增 `details` 分支：

```qml
} else if (appViewModel.currentView === "details") {
    appViewModel.mediaDetailsBack()
}
```

从继续观看等没有媒体库上下文的入口进入详情页时，`mediaDetailsBack()` 会回退到 `backToHome()`，保持原有行为。

## 没有改动的内容

本次没有修改：

- Emby API 路径。
- Jellyfin API 路径。
- 媒体库、媒体项、详情页的网络请求参数。
- `PlayerController`。
- `MpvVideoItem`。
- mpv `loadfile` 参数。
- native window、DPR、geometry 和 Z 序逻辑。
- WebDAV / SMB / IPTV 逻辑。

本地构建通过，包含 QML cache 生成、moc / QML 类型注册和 C++ 链接。

需要人工点测：

- Emby 媒体库 -> 影片详情 -> 返回：回到当前媒体库列表。

- Emby 继续观看 -> 详情 -> 返回：仍回到服务首页。
- Jellyfin 继续观看 -> 详情 -> 返回：仍回到服务首页。
- 播放退出回到详情后，再点击返回：按详情来源回到媒体库或服务首页。
