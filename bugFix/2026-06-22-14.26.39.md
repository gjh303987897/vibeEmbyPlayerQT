# WebDAV 实现后的问题修复回顾

## 背景

完成 WebDAV 第一版功能后，先后在启动、WebDAV 目录浏览、WebDAV 视频播放三个环节发现问题。本文件记录每个问题的现象、原因、修复内容和修复后的验证情况，方便后续回顾。

## 问题一：启动时报缺少 `Qt6Xmld.dll`

### 现象

启动程序时报错：

```text
由于找不到 Qt6Xmld.dll，无法继续执行代码。重新安装程序可能会解决此问题。
```

### 原因

- WebDAV 客户端新增了 XML 解析，工程链接了 `Qt6::Xml`。
- 当前项目没有自动执行完整 `windeployqt`。
- CMake 原本只在 Windows post-build 中复制 `libmpv-2.dll`，没有复制新增的 Qt XML 运行时 DLL。
- 因此编译通过，但直接运行 `build-clang/vibePlayerQT.exe` 时找不到 `Qt6Xmld.dll`。

### 修复

- 修改 `CMakeLists.txt`。
- 在 Windows post-build 步骤中增加：
  - `$<TARGET_FILE:Qt6::Xml>`
  - `$<TARGET_FILE_NAME:Qt6::Xml>`
- 构建后自动复制 `Qt6Xmld.dll` 到 `build-clang` 输出目录。

### 验证

- 重新执行 `.\scripts\build-clang.cmd`，构建通过。
- 确认 `D:\workspace\vibePlayerQT\build-clang\Qt6Xmld.dll` 已存在。
- 重新启动程序后不再出现缺 DLL 报错。

## 问题二：rclone WebDAV 服务目录显示为空

### 现象

用户添加名为 `AliCloud-Rclone-Onedrive-WebDAV` 的 WebDAV 服务后，进入服务页面显示文件夹为空，但实际 WebDAV 目录中存在文件。

### 原因

- WebDAV `PROPFIND` 请求本身能成功返回。
- 原解析逻辑使用了：

```cpp
document.elementsByTagNameNS(QStringLiteral("*"), QStringLiteral("response"))
```

- Qt DOM 对 `"*"` 作为 namespace 通配符的兼容并不稳。
- rclone/OneDrive WebDAV 返回的 XML 通常带 `DAV:` namespace。
- 结果是响应 XML 存在，但没有解析到任何 `response` 节点，最终 UI 显示为空。

### 修复

- 在 `WebDavClient.cpp` 中新增基于 `localName` 的递归节点查找。
- 不再依赖具体 namespace 前缀或 namespace 通配符。
- 解析 `propstat` 时优先选择 HTTP 200 的属性块。
- 当 `PROPFIND` 成功但解析结果仍为空时，写入安全日志：
  - 记录 URL
  - 记录响应字节数
  - 不记录用户名、密码或敏感头信息

### 验证

- 重新构建通过。
- 重新启动程序。
- 新版会更兼容 rclone 这类带 DAV namespace 的响应。

## 问题三：WebDAV 视频播放时报 `Playback failed: loading failed`

### 现象

WebDAV 文件列表可以正常打开，网页中使用 WebDAV 播放同一视频也正常，但在本播放器中点击视频播放时报：

```text
Playback failed: loading failed
```

日志中进一步显示：

```text
[error/curl] TLS certificate verification failed.
[error/stream] Failed to open https://...
```

### 原因

- WebDAV 文件列表由 Qt 网络层请求。
- Qt 网络层已经支持“允许自签名证书”，所以目录浏览可以成功。
- 视频播放交给 libmpv。
- libmpv 使用自己的 curl/TLS 校验逻辑，不会自动继承 Qt 网络层的证书确认结果。
- 因此即使 WebDAV 服务允许自签名证书，libmpv 播放时仍然因为 TLS 校验失败而无法打开视频。

### 修复

- 在 `AppViewModel` 中新增播放期临时属性：
  - `playbackAllowInsecureTls`
- 在 WebDAV 视频播放时，如果当前 WebDAV 服务开启了允许自签名证书，则把该值设为 `true`。
- 在 `MpvVideoItem` 中新增：
  - `allowInsecureTls`
- 在 `PlayerController::playUrl()` 中新增参数：
  - `allowInsecureTls`
- 播放前根据该参数设置 libmpv：

```cpp
mpv_set_option_string(m_mpv, "tls-verify", allowInsecureTls ? "no" : "yes");
```

- 离开播放或清理当前播放状态时，清空该临时播放状态。

### 验证

- 重新执行 `.\scripts\build-clang.cmd`，构建通过。
- 重新启动程序，进程正常保持运行。
- 日志确认此前失败原因为 libmpv TLS 校验失败，修复后新版已把 WebDAV 自签名证书设置传递到 libmpv。

## 当前注意事项

- `tls-verify=no` 只在当前 WebDAV 服务允许自签名证书时启用。
- 其他媒体源和未允许自签名证书的 WebDAV 服务仍保持 TLS 校验。
- WebDAV 密码仍不写入 SQLite，继续通过系统凭据库或临时输入密码处理。
