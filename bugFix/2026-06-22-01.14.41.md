# 修复 IPTV 添加时报 `servers.username` 非空约束失败

## 问题现象

添加新的 IPTV 服务时，保存失败并提示：

```text
NOT NULL constraint failed: servers.username 无法获取行
```

## 原因分析

- `servers.username` 数据库字段定义为 `TEXT NOT NULL DEFAULT ''`。
- IPTV 服务本身不需要用户名，因此 `AppViewModel::makeServerConfig()` 中给 IPTV 生成的 `ServerConfig.username` 使用了 `QString {}`。
- `QString {}` 是 null QString，绑定到 SQLite 参数时会以 `NULL` 写入，而不是普通空字符串。
- SQLite 因此触发 `servers.username` 的 `NOT NULL` 约束失败。

## 修复内容

- 在 `src/database/SessionRepository.cpp` 中新增 `nonNullText()`，用于将 null QString 统一转换为真正的空字符串。
- 在保存服务的通用 `upsertServer()` 中，对 `:username` 绑定值使用 `nonNullText(server.username)`。
- 在 `src/viewmodels/AppViewModel.cpp` 的 IPTV `ServerConfig` 构造处，将用户名从 `QString {}` 改为 `QStringLiteral("")`，避免 IPTV 配置层继续产生 null QString。

## 影响范围

- IPTV 添加时不再因为无用户名触发 `servers.username` 的非空约束。
- Emby/Jellyfin 仍使用原有用户名逻辑，不改变登录、保存和自动登录流程。
- 数据库表结构不变，只修复写入参数。

## 验证

- 已执行 `.\scripts\build-clang.cmd`。
- 首次构建失败原因是运行中的 `vibePlayerQT.exe` 占用了输出文件。
- 结束占用进程后重新构建，构建通过。
