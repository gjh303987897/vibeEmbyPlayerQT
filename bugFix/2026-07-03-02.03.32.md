# 修复继续观看列表退出播放后仍显示旧进度

## 问题现象

用户进入 Emby 后，从继续观看 / 历史记录中的某个剧集开始播放。

初始卡片显示的观看进度是旧位置，例如 12:00。播放到约 24:00 后退出播放器，继续观看列表仍然显示旧位置；但点击该记录进入详情页后，详情页显示的是正确的新进度。

复核 Jellyfin 后确认，同一类问题也可能出现在 Jellyfin：播放状态上报和详情页拉取是正确的，但首页继续观看列表如果不做本地同步，也可能继续显示旧的列表模型数据。

## 原因分析

播放停止时，应用已经调用 Emby / Jellyfin 的播放上报接口：

- `/Sessions/Playing/Progress`
- `/Sessions/Playing/Stopped`

上报体中的 `PositionTicks` 会告知服务端当前播放位置。

但本地 UI 中的 `continueItems` 列表模型没有同步更新：

- 继续观看卡片使用的是进入首页时拉取到的 `MediaItem.playbackPositionTicks` / `playedPercentage`。
- 退出播放后只发出了服务端上报，没有更新本地列表模型。
- 详情页之所以正确，是因为点击卡片后会重新调用 `fetchItemDetails()`，从服务端重新拉取 `UserData.PlaybackPositionTicks` 和 `UserData.PlayedPercentage`。

因此实际问题不是服务端没有更新，而是本地继续观看列表没有在播放上报后刷新或合并最新进度。

## 官方文档复查

复查了 Emby 播放状态 API 文档：

- `PositionTicks` 是播放进度上报字段。
- 服务端详情和列表数据会通过用户数据中的播放位置与播放百分比返回当前进度。

同时复查了 Jellyfin 官方 OpenAPI：

- `/UserItems/Resume` 用于获取继续观看列表。
- `/Items/{itemId}` 用于获取单个媒体详情。
- `/Sessions/Playing/Progress` 的 `PlaybackProgressInfo` 请求体包含 `ItemId`、`MediaSourceId`、`PlaySessionId`、`IsPaused` 和 `PositionTicks`。
- `/Sessions/Playing/Stopped` 的 `PlaybackStopInfo` 请求体包含 `ItemId`、`MediaSourceId`、`PlaySessionId` 和 `PositionTicks`。

当前 `JellyfinClient` 使用的路径与上述官方 OpenAPI 一致，因此本次修复不需要改 Jellyfin API 路径，而是修复共享 ViewModel/ListModel 的本地进度同步。

参考：

- https://dev.emby.media/reference/RestAPI/PlaystateService/postSessionsPlayingProgress.html
- https://repo.jellyfin.org/releases/openapi/jellyfin-openapi-stable.json

## 修复内容

### 1. MediaItemListModel 支持局部更新播放进度

在 `MediaItemListModel` 中新增：

```cpp
bool updatePlaybackProgress(const QString& itemId,
                            qint64 playbackPositionTicks,
                            double playedPercentage,
                            bool played);
```

该方法按 `itemId` 查找列表项，只更新对应行的：

- `playbackPositionTicks`
- `playedPercentage`
- `played`

并发出对应行的 `dataChanged()`，避免重置整个列表。

同时新增 QML role：

```text
playbackPositionTicks
```

方便后续 UI 如需展示精确断点位置时直接读取。

### 2. AppViewModel 在播放上报后同步本地模型

新增：

```cpp
applyReportedPlaybackProgress(...)
mergeRecentPlaybackProgress(...)
```

播放进度上报和播放停止上报后会立即同步：

- 当前详情项 `m_selectedItem`
- 首页继续观看列表 `m_continueItems`
- 当前媒体库列表 `m_items`
- 剧集列表 `m_seriesEpisodes`

这样退出播放后，首页继续观看卡片不会继续显示旧进度。

### 3. 防止服务端短暂旧数据覆盖本地新进度

播放停止后会延迟刷新一次继续观看列表。

由于服务端写入播放状态可能是异步的，刚刷新时有机会仍返回旧的 `UserData`。为避免把本地 24:00 又覆盖回 12:00，`AppViewModel` 会保留最近 30 秒内刚上报的播放进度快照，并在刷新继续观看列表时合并到服务端返回结果中。
